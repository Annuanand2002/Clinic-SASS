How workflows work

What this is

Workflows define the steps a complaint or appointment can go through. Each record has a current step on the active workflow. Staff see allowed next steps; admins can move the record forward when rules allow.

One workflow per type

There is at most one workflow for complaints and one for appointments. Only one workflow per type can be active at a time; activating a workflow turns off the previous one for that type.

Building the graph

  • Start — entry point (one per workflow).
  • State — a stage (e.g. “Submitted”, “In review”).
  • Action — a step that represents something to do.
  • Decision — branching (conditions on outgoing connections).
  • End — terminal step.

Connecting nodes

Click the blue port on the right of a node, then the green port on the left of another node to create a connection. Pan the canvas by dragging empty space; use the mouse wheel to zoom.

Labels & conditions

Select a connection to set an optional edge label (shown on the canvas) and condition JSON for advanced rules. Save the workflow when you are finished editing.

Day-to-day use

On a complaint or appointment, the Workflow panel shows the current step, possible next steps, and history. Elevated roles can transition to the next step; others can follow progress in read-only mode.
